# 核心知识

## 代码审计基础知识

### 什么是代码审计
代码审计 (Code Auditing) 本质上是检查源代码以发现潜在的缺陷、漏洞或不符合安全标准的行为。它可以手动执行，也可以借助自动化工具。

- 手动审计：由安全专家逐行审查代码，寻找逻辑漏洞、权限问题、不安全的 API 调用等。优点是深入、全面；缺点是耗时、成本高。
- 自动化审计：使用工具（如使用的 npm audit）来扫描代码库，自动识别已知的安全漏洞和依赖项风险。优点是快速、高效、可重复；缺点是可能存在误报，且无法发现未知的、复杂的逻辑漏洞。  

项目将聚焦于自动化审计中的开源依赖项审计，这是识别项目中使用到的第三方库是否存在已知漏洞的关键步骤。

## npm audit
>`npm audit` 是 npm 官方提供的安全审计命令，它会扫描你项目中的所有依赖项，并与 npm 官方维护的漏洞数据库进行比对，以识别潜在的安全风险。  

### npm audit 运行原理

1. 收集依赖信息：当你运行 npm audit 时，npm 会读取你的 package.json 文件中的 dependencies 和 devDependencies，并根据 package-lock.json（或 npm-shrinkwrap.json）文件中的确切版本信息，构建一个完整的依赖树。
2. 发送审计请求：npm CLI (命令行接口) 会将这个依赖树的元数据（包名、版本号等）发送到 https://registry.npmjs.org/ 的审计 API 端点。
3. 漏洞数据库比对：npm 官方的服务器收到请求后，会将其与一个名为 Node Security Platform (NSP) 的漏洞数据库进行比对。该数据库包含了已知的、针对特定包和版本号的安全漏洞。
4. 返回审计报告：服务器将比对结果返回给你的终端。这个结果就是你看到的审计报告，它会列出所有发现的漏洞，并提供漏洞详情、严重等级以及修复建议。

### 审计结果解读
`npm audit` 的输出通常包含以下几个关键部分：

- 漏洞摘要：显示审计发现的漏洞总数，并按严重等级（High、Moderate 等）进行分类。
- 严重等级 (Vulnerability Levels)：这是衡量漏洞风险的关键指标。npm audit 沿用行业的标准分类：
  - Low（低）：漏洞可能导致轻微的风险，通常不直接影响应用程序功能或数据安全。
  - Moderate（中）：漏洞可能导致一定风险，例如拒绝服务攻击或数据泄露，但利用难度较高。
  - High（高）：漏洞可能导致严重风险，例如远程代码执行、敏感数据大规模泄露，且利用难度相对较低。
  - Critical（严重）：最危险的漏洞，可导致远程代码执行、权限提升等，对系统造成毁灭性打击，且通常非常容易被利用。

- 漏洞详情：为每一个漏洞提供详细信息，包括：
  - Vulnerability：漏洞名称和描述。
  - Package：受影响的依赖包。
  - Patched in：提供修复此漏洞的最低安全版本号。
  - Dependency of：列出受影响的包是在哪个父依赖中被引入的。
  - Path：显示从你的项目到受影响包的完整依赖链，这对于理解漏洞是如何引入的至关重要。
